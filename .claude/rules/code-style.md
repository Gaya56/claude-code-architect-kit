## Code Style

- Use clear, descriptive names. Avoid abbreviations unless universally understood.
- Functions should do one thing. If a function needs a comment explaining what it does, it should probably be split.
- Prefer returning early over deep nesting.
- Error messages should be actionable -- tell the user what went wrong AND what to do about it.
- Don't add comments that repeat the code. Only comment the "why", not the "what".
